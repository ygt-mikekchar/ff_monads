Overview

Represents a value for an operation that may fail. Result is normally used in conjunction with â€˜.map` or `.and_then` to operate on a value where you want to concentrate on the success path. If the operation fails, the computation can drop out to return a failure.

Examples:

You can mix in this module.

include FFMonads::Result::Mixin

A value from a successful operation.

success(42)

A value from a failed operation.

failure("divide by zero")

#and_then(&block) ⇒ FFMonads::Maybe

Applies an potentially failed value to a block. It takes the value from the Result, transforms it, and returns it without putting it back into a Result. It assumes the block will put the value back into a Result.

Examples:

â€˜.and_then` allows you to choose the Result to return.

# This returns failure
success(0).and_then do |x|
  if x == 0
    failure("divide by zero")
  else
    success(100 / x)
  end
end

#map(&block) ⇒ FFMonads::Result

Applies a potentially failed value to a block. It takes the value from the Result, transforms it, puts back into a Result and returns it.

Examples:

Called with success, it will always return success.

success(42).map {|x| x + 1} # returns success(43)

Called with failure, it will always return failure.

failure("divide by zero").map {|x| x + 1} # returns failure("divide by zero")
