Overview

Represents an optional value. Maybe is normally used in conjunction with ‘.map` or `.and_then` to operate on an optional value without crashing when the value does not exist.

Examples:

You can mix in this module.

include FFMonads::Maybe::Mixin

some(42)

a value that does not exist.

#and_then(&block) ⇒ FFMonads::Maybe

Applies an optional value to a block. It takes the value from the Maybe, transforms it, and returns it without putting it back into a Maybe. It assumes the block will put the value back into a Maybe.

Examples:

‘.and_then` allows you to choose the Maybe to return.

# This returns none
some(0).and_then do |x|
  if x == 0
    none
  else
    some(100 / x)
  end
end

#map(&block) ⇒ FFMonads::Maybe

Applies an optional value to a block. It takes the value from the Maybe, transforms it, puts back into a Maybe and returns it.

Examples:

Called with some, it will always return some.

some(42).map {|x| x + 1} # returns some(43)

Called with none, it will always return none.

none.map {|x| x + 1} # returns none
